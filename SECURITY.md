# Security Policy

We take the security of **OpenDX-Lab** very seriously. If you discover any security vulnerabilities, please report them promptly so we can address them in a timely manner.

## Supported Versions

We currently provide security patches only for the version under active development:

| Version | Supported | Notes |
|---------|-----------|-------|
| 1.0.x-dev | ✅ Yes | Version being prepared for OLP Open Source 2026 |
| < 1.0.0 | ❌ No | Older experimental versions |

## Reporting a Vulnerability

**Please DO NOT disclose security vulnerabilities through public GitHub Issues.**

To report a vulnerability:
1. Send an email directly to: **thanhheo7749@gmail.com**
2. In your email, please include:
   * The type of vulnerability (e.g., XSS, SQL Injection, RCE, etc.).
   * The affected services (e.g., Keycloak SSO, Next.js Dashboard, PostgreSQL, etc.).
   * Step-by-step instructions to reproduce the vulnerability (Proof of Concept).
   * Your assessment of the vulnerability's impact on the system.

We will acknowledge your email within **48 hours** and respond with a remediation plan and timeline for a fix.

Thank you for helping keep OpenDX-Lab secure!
